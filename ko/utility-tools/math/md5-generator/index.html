해시 변형 #설명 변형
MD5-32
표준 출력 RFC 1321에 정의된 표준 MD5 형식은 모든 길이의 입력에 대해 128비트 다이제스트를 생성하고 이를 32 소문자 16진수 문자로 표시합니다. 파일 무결성 검사, 데이터 지문 및 중복 제거에 널리 사용됩니다. 예: 5d41402abc4b2a76b9719d911017c592
MD5-16
잘린 출력 표준 MD5-32 출력(총 16자)의 9–24 문자를 가져와 중간 64비트를 효과적으로 노출합니다. 더 짧지만 충돌 위험이 더 높습니다(약 2⁻⁶⁴ ). 보안 수준이 낮은 시나리오의 짧은 ID 또는 토큰에 적합하지만 보안 유효성 검사에는 권장되지 않습니다.
MD5-32 대문자
대문자 출력 MD5-32와 정확히 동일한 해시 값이지만 대문자( A–F )입니다. 일부 레거시 시스템, 결제 서명 또는 엔터프라이즈 인터페이스에는 대문자 출력이 필요합니다. 예: 5D41402ABC4B2A76B9719D911017C592
📥
입력 인코딩 #인코딩 설명
UTF-8 텍스트
기본값 입력을 UTF-8 인코딩된 문자열로 처리하고 해싱 전에 중국어, 일본어 또는 이모티콘과 같은 멀티바이트 문자를 바이트 시퀀스로 자동 변환합니다. 이는 가장 일반적인 모드이며 일반 텍스트 입력에 작동합니다.
16진수 바이트 스트림
16진수 입력을 16진수 문자열( 0-9, a-f , 공백은 무시됨)로 처리합니다. 해싱 전에 두 문자마다 1바이트로 구문 분석됩니다. 이는 키나 프로토콜 페이로드와 같은 원시 바이너리 데이터를 해싱할 때 유용합니다. 입력 예: 48 65 6c 6c 6f
Base64
Base64 디코드 입력을 표준 Base64 문자열로 처리하고 이를 원시 바이트로 디코딩한 후 해당 바이트를 해시합니다. Base64에 표시된 파일 또는 이미지 콘텐츠의 원본 해시를 계산할 때 유용합니다. 예: SGVsbG8=
📋
일괄 모드 #배치 설명
단일/전체 텍스트
기본값 줄바꿈을 포함한 전체 입력 상자 콘텐츠를 하나의 완전한 값 으로 해시하고 단일 결과를 출력합니다. 전체 텍스트, 단락 또는 파일 콘텐츠의 지문 채취에 적합합니다. 해시 입력에는 줄 바꿈이 포함됩니다.
라인별로 분할
일괄 \n 을 구분 기호로 사용하여 입력을 여러 줄로 분할하고 각 줄을 독립적으로 해시합니다. 빈 줄은 자동으로 무시됩니다. 기본 출력에는 한 줄에 하나의 해시가 표시되며 아래 전체 배치 테이블에는 MD5-32, MD5-16 및 원클릭 CSV 내보내기가 가능한 대문자 결과가 포함됩니다. 사용자 이름, 비밀번호 목록 및 데이터 세트 처리에 유용합니다.
