전문 WebSocket 테스트 도구 기능

한 번의 클릭으로 ws와 wss 사이를 전환하세요. 암호화된 일반 연결과 호환되는 자동 완성 URL 접두어입니다.

서버 에 따라 프로토콜 자동 선택

사용자 정의 헤더

인증, 토큰 또는 사용자 정의 매개변수에 대한 HTTP 헤더를 추가하세요. 핸드셰이크 중에 전송됩니다.

헤더 키/값 을 입력하려면 '+ 추가'를 클릭하세요.

실시간 통계

전송/수신 횟수, 총 트래픽 및 현재 대기 시간을 모니터링합니다. 매초마다 업데이트됩니다.

하트비트 핑을 통해 계산된 지연 시간

사전 설정 및 템플릿

즉각적인 테스트를 위한 내장형 공개 테스트 서버 및 메시지 템플릿(Ping, Hello, Subscribe 등).

카드를 클릭하여 URL을 자동으로 입력하거나 메시지를 보내세요.

강력한 로깅

유형 필터, 키워드 검색, 자동 스크롤 및 텍스트 파일로 내보내기 기능을 갖춘 실시간 로그입니다.

최근 500개의 메시지를 유지합니다

텍스트, JSON, 16진수. JSON 형식/축소 버튼, 빠른 전송을 위한 Ctrl+Enter 단축키.

자주 묻는 질문(FAQ)

웹소켓이란 무엇인가요?

WebSocket은 단일 TCP 연결을 통해 전이중 통신을 제공하는 프로토콜로, 서버가 클라이언트에 데이터를 푸시할 수 있도록 해줍니다. 이 도구는 WebSocket 서비스가 올바르게 작동하는지 테스트하는 데 도움이 됩니다.

WebSocket 연결을 어떻게 테스트하나요?

WebSocket 서버 URL(예: wss://echo.websocket.org)을 입력하고 '연결'을 클릭하세요. 연결되면 입력 영역에 메시지를 입력하고 '보내기'를 클릭하세요. 받은 메시지가 로그에 나타납니다.

왜 연결이 실패하나요?

가능한 이유: 잘못된 서버 주소, 서버가 WSS를 지원하지 않음(WS 시도), 방화벽 차단, 서버 다운 또는 CORS 제한. URL을 확인하고 서버가 WebSocket 연결을 허용하는지 확인하세요.

맞춤 헤더를 어떻게 추가하나요?

'연결 설정'에서 '+ 추가'를 클릭하여 헤더 이름과 값을 입력하세요. 이러한 헤더는 WebSocket 핸드셰이크 중에 전송되며 인증 또는 사용자 지정 매개변수에 유용합니다.

어떤 메시지 형식이 지원되나요?

텍스트, JSON 및 16진수. 형식을 선택하고 내용을 입력하면 자동으로 변환됩니다. JSON 모드에는 쉽게 편집할 수 있도록 형식화 및 축소 버튼이 포함되어 있습니다.

로그를 저장할 수 있나요?

네. 로그 영역의 '내보내기' 버튼을 클릭하면 현재 로그를 텍스트 파일로 다운로드할 수 있습니다.

지연 시간은 어떻게 계산되나요?

연결 후 도구는 내장된 핑 메시지를 10초마다 보내고 퐁이 수신될 때까지의 시간을 측정하여 현재 대기 시간을 표시합니다.
