🚀 Công cụ POST trực tuyến / Trình kiểm tra API

Công cụ gỡ lỗi yêu cầu HTTP giao diện người dùng thuần túy, hỗ trợ nhiều phương thức HTTP và định dạng dữ liệu

Triển khai giao diện người dùng thuần túy

Chạy hoàn toàn trên trình duyệt, không cần máy chủ. Tất cả các yêu cầu HTTP được gửi qua API tìm nạp của trình duyệt, đảm bảo quyền riêng tư và bảo mật dữ liệu.

Nhiều phương thức HTTP

Hỗ trợ POST, GET, PUT, DELETE, PATCH năm phương thức HTTP, đáp ứng nhu cầu phát triển API RESTful cơ bản.

Hỗ trợ dữ liệu đa định dạng

Hỗ trợ các định dạng dữ liệu JSON và biểu mẫu, thuận tiện cho việc thử nghiệm các loại giao diện API khác nhau. Trình soạn thảo JSON cung cấp tính năng đánh dấu và xác thực cú pháp.

Tiêu đề yêu cầu tùy chỉnh

Cấu hình linh hoạt các tiêu đề yêu cầu, hỗ trợ các tiêu đề phổ biến như Content-Type, Authorization, User-Agent, đáp ứng các yêu cầu kiểm tra API phức tạp.

Bản ghi lịch sử URL

Tự động lưu các URL được sử dụng gần đây, thuận tiện cho việc chuyển đổi nhanh và kiểm tra lặp lại, nâng cao hiệu quả gỡ lỗi.

Xem trước phản hồi theo thời gian thực

Hiển thị ngay lập tức mã trạng thái HTTP, tiêu đề phản hồi và nội dung phản hồi, hỗ trợ hiển thị định dạng JSON, dễ dàng phân tích kết quả trả về API.

Các câu hỏi thường gặp

Công cụ này có hỗ trợ HTTPS không?

Có, hỗ trợ đầy đủ giao thức HTTPS. Bạn có thể nhập bất kỳ URL HTTPS hợp lệ nào để kiểm tra. Công cụ này tuân theo các chính sách bảo mật của trình duyệt.

Làm cách nào để xử lý các yêu cầu có nguồn gốc chéo?

Các yêu cầu có nguồn gốc chéo yêu cầu máy chủ mục tiêu định cấu hình tiêu đề CORS (Chia sẻ tài nguyên nguồn gốc chéo). Nếu bạn gặp phải lỗi nhiều nguồn gốc, hãy đảm bảo máy chủ API đã định cấu hình chính sách CORS chính xác, chẳng hạn như Access-Control-Allow-Origin, v.v.

Nó có hỗ trợ tải tập tin lên không?

Phiên bản hiện tại chủ yếu hỗ trợ các định dạng dữ liệu biểu mẫu và JSON. định dạng multipart/form-data (để tải lên tệp) sẽ được hỗ trợ trong các phiên bản sau. Bạn có thể sử dụng API FormData với công cụ này để kiểm tra việc tải tệp lên.

Dữ liệu có được lưu vào máy chủ không?

Không. Tất cả dữ liệu được lưu trong bộ nhớ cục bộ của trình duyệt (localStorage), không được tải lên bất kỳ máy chủ nào, đảm bảo khóa API và bảo mật dữ liệu nhạy cảm của bạn.

Nó có hỗ trợ thử nghiệm WebSocket không?

Phiên bản hiện tại tập trung vào thử nghiệm yêu cầu HTTP/HTTPS. Công cụ kiểm tra WebSocket đang được phát triển và sẽ được cung cấp trong các phiên bản sau.

Nó có phù hợp để học giao thức HTTP không?

Hoàn hảo cho việc học tập! Công cụ này hiển thị trực quan tất cả các thành phần của yêu cầu và phản hồi HTTP, một công cụ lý tưởng để tìm hiểu giao thức HTTP, thiết kế API RESTful và kỹ thuật gỡ lỗi.

Mẹo sử dụng

  • Sử dụng phím tắt Ctrl+Enter (Windows) hoặc Cmd+Enter (Mac) để gửi nhanh yêu cầu
  • Dữ liệu JSON được định dạng tự động để dễ đọc và chỉnh sửa
  • Dữ liệu phản hồi có thể được sao chép vào bảng nhớ tạm để phân tích thêm
  • Để phát triển cục bộ, định dạng URL được đề xuất: http://localhost:port
  • Lưu cấu hình API được sử dụng thường xuyên để thử nghiệm lặp lại nhanh chóng
