Perguntas frequentes (FAQ)

O que é WebSocket?

WebSocket é um protocolo que fornece comunicação full-duplex em uma única conexão TCP, permitindo que os servidores enviem dados aos clientes. Esta ferramenta ajuda a testar se o seu serviço WebSocket funciona corretamente.

Como faço para testar uma conexão WebSocket?

Insira o URL do servidor WebSocket (por exemplo, wss://echo.websocket.org), clique em 'Conectar'. Uma vez conectado, digite uma mensagem na área de entrada e clique em ‘Enviar’. As mensagens recebidas aparecem no log.

Por que a conexão falha?

Possíveis motivos: endereço de servidor errado, servidor não suporta WSS (tente WS), bloqueio de firewall, servidor inativo ou restrições de CORS. Verifique a URL e certifique-se de que o servidor permite conexões WebSocket.

Como adiciono cabeçalhos personalizados?

Em 'Configurações de conexão', clique em '+ Adicionar' para inserir o nome e o valor do cabeçalho. Esses cabeçalhos são enviados durante o handshake do WebSocket, úteis para autenticação ou parâmetros personalizados.

Quais formatos de mensagem são suportados?

Texto, JSON e hexadecimal. Selecione o formato, insira seu conteúdo e ele será convertido automaticamente. O modo JSON inclui botões de formatação e redução para fácil edição.

Posso salvar o log?

Sim. Clique no botão 'Exportar' na área de log para baixar o log atual como um arquivo de texto.

Como a latência é calculada?

Após a conexão, a ferramenta envia uma mensagem de ping integrada a cada 10 segundos e mede o tempo até o recebimento do pong, exibindo a latência atual.
