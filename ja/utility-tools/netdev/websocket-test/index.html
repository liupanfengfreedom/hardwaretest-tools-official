よくある質問 (FAQ)

WebSocketとは何ですか?

WebSocket は、単一の TCP 接続上で全二重通信を提供するプロトコルであり、サーバーがクライアントにデータをプッシュできるようにします。このツールは、WebSocket サービスが正しく動作するかどうかをテストするのに役立ちます。

WebSocket 接続をテストするにはどうすればよいですか?

WebSocket サーバーの URL (例: wss://echo.websocket.org) を入力し、[接続] をクリックします。接続したら、入力エリアにメッセージを入力し、「送信」をクリックします。受信したメッセージはログに表示されます。

接続が失敗するのはなぜですか?

考えられる理由: サーバー アドレスが間違っている、サーバーが WSS をサポートしていない (WS を試してください)、ファイアウォールのブロック、サーバーのダウン、または CORS の制限。 URL を確認し、サーバーが WebSocket 接続を許可していることを確認してください。

カスタム ヘッダーを追加するにはどうすればよいですか?

[接続設定] で [+ 追加] をクリックしてヘッダー名と値を入力します。これらのヘッダーは WebSocket ハンドシェイク中に送信され、認証やカスタム パラメーターに役立ちます。

どのようなメッセージ形式がサポートされていますか?

テキスト、JSON、および 16 進数。形式を選択し、コンテンツを入力すると、自動的に変換されます。 JSON モードには、編集を容易にするフォーマット ボタンと縮小ボタンが含まれています。

ログを保存できますか?

はい。ログ領域の「エクスポート」ボタンをクリックして、現在のログをテキスト ファイルとしてダウンロードします。

レイテンシーはどのように計算されますか?

接続後、ツールは組み込みの ping メッセージを 10 秒ごとに送信し、ポンを受信するまでの時間を測定し、現在の遅延を表示します。
