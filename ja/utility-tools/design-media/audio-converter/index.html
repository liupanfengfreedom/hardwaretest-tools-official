プロフェッショナル オーディオ コンバーターの機能

🎯 ロスレスおよびロッシーのサポート

WAV、AIFF ロスレス形式、または MP3、OGG、WebM ロッシー形式から選択します。アーカイブ、共有、モバイルデバイスに最適です。

MP3 ビットレートは最大 320kbps、ほぼロスレスの品質。

🎬 ビデオから音声を抽出

MP4、MKV、MOV、AVI ファイルをドラッグすると、選択した形式でオーディオ ストリームが自動的に抽出されます。追加の工具は必要ありません。

一般的なビデオ形式をサポートします。デコードが失敗した場合は、まずビデオを再エンコードしてみてください。

⚙️ 細かいパラメータ制御

サンプル レート (8kHz ～ 96kHz)、チャンネル数 (モノラル/ステレオ)、ビットレート (MP3) をカスタマイズします。出力品質を完全に制御します。

元のファイルのサンプル レートとチャンネルを維持するには、「自動」を選択します。

🔒 プライバシー保護、アップロード不要

すべての処理はブラウザ内で行われます。ファイルがデバイスの外に出ることはありません。機密性の高い録音を自信を持って変換します。

プライバシーの心配はありません – 変換して削除します。

📦 バッチ変換とキュー

複数のファイルを一度に選択します。それらはキューに並び、1 つずつ処理されます。後で個別にダウンロードするか、バッチパッケージをダウンロードします。

いつでもリストをクリアして最初からやり直すことができます。

🎧 内蔵ミニプレーヤー

ダウンロードする前に、変換されたオーディオをページ上で直接プレビューできます。時間を節約し、繰り返しのダウンロードを回避します。

一時停止、シーク - クイック プレビュー。

よくある質問 (FAQ)

どのオーディオ形式がサポートされていますか?

現在、MP3、WAV、FLAC、AAC (M4A)、OGG (Opus)、AIFF、WMA、WebM (Opus) 間の変換をサポートしています。ビデオ ファイル (MP4、MKV、MOV、AVI、3GP、TS) からオーディオを抽出し、これらの形式のいずれかに変換することもできます。時間の経過とともにさらに多くの形式が追加される予定です。

変換された音声の品質はどのくらいですか?

品質は設定に完全に依存します:
- WAV または AIFF の場合: 元のオーディオ データと同一のロスレス PCM。
- MP3 の場合: スライダーを使用してビットレート (64 kbps ～ 320 kbps) を設定します。高いほど品質が向上し、ファイルが大きくなります。
- サンプル レート オプションを使用すると、ダウンサンプリングしてファイル サイズを削減できます (例: 48kHz → 44.1kHz または 22kHz)。
最良の結果を得るには、元のサンプル レートとステレオを維持することをお勧めします。

本当にファイルがアップロードされていないのでしょうか？

はい、100% ローカル処理です。デコードには Web Audio API、MP3 のエンコードには lamejs、Opus には MediaRecorder を使用します。すべてはブラウザのメモリ内で行われ、ネットワーク転送は行われません。このツールはオフラインでも (ページが一度読み込まれた後は) 使用できます。

一部のビデオ ファイルを変換できないのはなぜですか?

ブラウザは、サポートしているビデオ コーデックのみをデコードできます。たとえば、一部の MKV ファイルには HEVC (H.265) ビデオが含まれている場合があり、Safari または Chrome はそのオーディオ トラックをサポートしていない可能性があります。より一般的なコーデック (H.264 + AAC など) を使用してみてください。問題が解決しない場合は、まずビデオ エンコーディングを変換してから、オーディオを抽出します。

複数のファイルを一度に変換できますか?

はい。いくつかのファイルを選択すると、キューに表示されます。 「変換開始」をクリックすると、ツールがそれらを 1 つずつ処理します。変換中にファイルを追加または削除できますが、現在処理中のファイルは変更できません。

大きなファイルを変換するとページがフリーズしますか?

オーディオ エンコードは、特に大きなファイルや高ビットレートの場合、CPU を集中的に使用します。可能な限り最適化していますが、メインスレッドが一時的にブロックされる場合があります。非常に大きなファイル (100 MB を超えるなど) の場合は、しばらくお待ちください。進行状況バーはリアルタイムで更新されます。将来のバージョンでは、パフォーマンスをよりスムーズにするために Web Workers が導入される可能性があります。

🔒 すべての処理はブラウザ内でローカルに実行されます。ファイルはサーバーにアップロードされません。
