プロフェッショナル向けマウス イベント周波数テスターの機能

リアルタイム周波数モニタリング

現在のマウス イベントの周波数 (Hz) をリアルタイムで表示します。ブラウザがマウス移動イベントを処理する実際の速度を理解するのに役立ちます。

注: Web 環境はブラウザとディスプレイのリフレッシュ レートによって制限されます

ジッタ解析

マウス イベントの時間安定性を反映して、イベント間隔の標準偏差を計算します。ジッター値が低いほど、イベントがより安定していることを示します。

ジッターが高いと、マウスの動きが滑らかでなくなる場合があります

最大記録周波数

テスト中に達成された最高のイベント頻度を自動的に記録し、システムの最高のパフォーマンスを把握するのに役立ちます。

Web 環境では通常、実際の USB ポーリング レートに達することができません

平均周波数の計算

履歴データに基づいて平均イベント頻度を計算し、より安定したパフォーマンス参照指標を提供します。

テスト時間を長くすると、より正確な平均値が得られます

リアルタイムチャート表示

周波数変化の傾向を動的な折れ線グラフで可視化し、性能の変動や異常を容易に観察できます。

グラフには、過去 100 データ ポイントの傾向が表示されます

CSV データのエクスポート

詳細な分析とデータ保存のために、CSV 形式での完全なテスト ログのエクスポートをサポートします。

エクスポートされたデータは、Excel、Python などでさらに分析するために使用できます。

よくある質問 (FAQ)

Web テストでは、実際の USB ポーリング レートよりも低いマウス イベント頻度が示されるのはなぜですか?

ブラウザ環境のマウス イベントは、ブラウザ イベントのスロットリング、ディスプレイのリフレッシュ レートの同期、オペレーティング システムのスケジュールなど、複数のレイヤーによって制限されます。Web ページは、ブラウザが提供するマウス イベント インターフェイスを介してのみ、USB HID レイヤーからの生データに直接アクセスできません。

より正確なテスト結果を得るにはどうすればよいですか?

1. ブラウザ ウィンドウが最前面にあり、アクティブになっていることを確認します
2. 不要なタブやアプリケーションを閉じる
3. テスト領域内で高速で連続的なマウス移動を実行します
4. の詳細なユーザー ガイド でその他のヒントをご覧ください

ジッターとは何ですか?なぜ重要なのでしょうか?

ジッターは、マウス イベントの間隔時間の安定性を反映します。ジッターが低いため、イベントの間隔が均一になり、スムーズなマウスの動きを実現します。平均周波数が高くても、ジッターが高いと、マウスの動きが途切れたり、不安定になったりすることがあります。

ブラウザごとに異なるテスト結果が表示されるのはなぜですか?

ブラウザー (Chrome、Firefox、Edge、Safari) が異なれば、使用するイベント処理メカニズム、スロットリング戦略、パフォーマンスの最適化も異なります。通常、Chrome はより高いイベント頻度を提供しますが、特定のパフォーマンスはオペレーティング システムとハードウェア構成にも依存します。

マウス イベントのパフォーマンスが正常かどうかを確認するにはどうすればよいですか?

通常の状況:
- 60Hz 表示: イベント周波数 ≈ 60Hz
- 120Hz 表示: イベント周波数 ≈ 120Hz
- 144Hz 表示: イベント周波数 ≈ 120-144Hz
- 240Hz 表示: イベント周波数 ≈ 200Hz
周波数がディスプレイのリフレッシュ レートよりも大幅に低い場合、パフォーマンスの問題が発生する可能性があります。

マウス イベントの頻度をいつテストすればよいですか?

推奨されるテスト状況:
- マウスの動きが途切れたり、反応しなくなったりする
- 異なるブラウザまたはシステム間の入力パフォーマンスの比較
- 新しいディスプレイまたはグラフィックス カードが入力遅延に与える影響の評価
- Web アプリケーションのマウス応答の問題の診断

⚠️

Web ポーリング レート テストに関する重要な注意事項

テスト結果を正しく理解するために、次の注意事項をよくお読みください

このページに表示される「ポーリング レート (Hz)」は、マウスの実際の USB ポーリング レートと同等ではありません。

🤔 Web テストの結果が通常 60Hz / 120Hz しかないのはなぜですか?

Webブラウザ環境の場合：

  • マウス データは ブラウザ イベント (mousemove / pointermove) を通じて取得されます
  • ブラウザ スロットルとマージ イベント
  • イベントのトリガー頻度は通常 画面のリフレッシュ レートと同期します
🖥️ ディスプレイのリフレッシュ レートと Web での観察可能な周波数
ディスプレイ リフレッシュ レート Web で監視可能なイベントの頻度
60Hz ≒ 60Hz
120Hz ≒ 120Hz
144Hz ≈ 120-144Hz
240Hz ≈ 200Hz

マウス ハードウェアが 1000Hz USB ポーリング レートに設定されている場合でも、
Web 環境は、USB HID レイヤーから実際のデータに直接アクセスできません。

本当の 1000Hz はどこに反映されますか?

実際の USB ポーリング レートは次のとおりです:

  • • オペレーティング システムの生入力 / HID レイヤー
  • • ゲーム エンジン (Unreal Engine、Unity など)
  • • デスクトップ レベルのテスト ツール (Web ベースではない)

🔧
デスクトップ ツールが必要な理由は何ですか?

Web テストはブラウザとオペレーティング システムによって制限されており、USB HID レイヤーから実際のポーリング データにアクセスできません。デスクトップ ツールは、システム レベルの API を通じてマウスの生の入力を直接キャプチャし、実際の USB ポーリング レート データを提供します。
