本页面显示的"回报率（Hz）并不等同于鼠标的真实 USB 回报率"。

🤔 为什么网页测试结果通常只有 60Hz / 120Hz？

在 Web 浏览器环境中：

  • 鼠标数据是通过 浏览器事件（mousemove / pointermove） 获取的
  • 浏览器会对事件进行 节流与合并
  • 事件触发频率通常 与屏幕刷新率同步
🖥️ 显示器刷新率与网页观测频率对比
显示器刷新率 网页可观测事件频率
60Hz ≈ 60Hz
120Hz ≈ 120Hz
144Hz ≈ 120–144Hz
240Hz ≈ 200Hz 左右

即使你的鼠标硬件设置为 1000Hz USB Polling Rate，
网页环境也无法直接访问 USB HID 层的真实数据。

那真实的 1000Hz 在哪里体现？

真实的 USB 回报率存在于：

  • • 操作系统的 Raw Input / HID 层
  • • 游戏引擎（如 Unreal Engine、Unity）
  • • 桌面级测试工具（非网页）
✅
本工具适合用来做什么？
  • • 检测鼠标在浏览器中的输入是否流畅
  • • 对比不同浏览器 / 显示器下的事件表现
  • • 检查是否存在卡顿、事件丢失或异常节流
❌
本工具不适合用来做什么？
  • • 判断鼠标是否支持 500Hz / 1000Hz
  • • 替代桌面级 USB Polling 测试工具
💡
建议

如果你需要验证鼠标的真实 USB 回报率，
请使用厂商驱动或桌面级 Raw Input 测试软件。

📌

提示：

请确保浏览器窗口处于前台、标签页为激活状态，否则浏览器会主动降低事件频率。
