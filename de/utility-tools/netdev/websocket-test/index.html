Häufig gestellte Fragen (FAQ)

Was ist WebSocket?

WebSocket ist ein Protokoll, das Vollduplex-Kommunikation über eine einzelne TCP-Verbindung ermöglicht und es Servern ermöglicht, Daten an Clients zu übertragen. Mit diesem Tool können Sie testen, ob Ihr WebSocket-Dienst ordnungsgemäß funktioniert.

Wie teste ich eine WebSocket-Verbindung?

Geben Sie die WebSocket-Server-URL ein (z. B. wss://echo.websocket.org) und klicken Sie auf „Verbinden“. Sobald die Verbindung hergestellt ist, geben Sie eine Nachricht in den Eingabebereich ein und klicken Sie auf „Senden“. Empfangene Nachrichten werden im Protokoll angezeigt.

Warum schlägt die Verbindung fehl?

Mögliche Gründe: falsche Serveradresse, Server unterstützt WSS nicht (versuchen Sie es mit WS), Firewall-Blockierung, Serverausfall oder CORS-Einschränkungen. Überprüfen Sie die URL und stellen Sie sicher, dass der Server WebSocket-Verbindungen zulässt.

Wie füge ich benutzerdefinierte Header hinzu?

Klicken Sie unter „Verbindungseinstellungen“ auf „+ Hinzufügen“, um den Namen und Wert des Headers einzugeben. Diese Header werden während des WebSocket-Handshakes gesendet und sind nützlich für die Authentifizierung oder benutzerdefinierte Parameter.

Welche Nachrichtenformate werden unterstützt?

Text, JSON und Hex. Wählen Sie das Format aus, geben Sie Ihren Inhalt ein und er wird automatisch konvertiert. Der JSON-Modus umfasst Formatierungs- und Minimierungsschaltflächen für eine einfache Bearbeitung.

Kann ich das Protokoll speichern?

Ja. Klicken Sie im Protokollbereich auf die Schaltfläche „Exportieren“, um das aktuelle Protokoll als Textdatei herunterzuladen.

Wie wird die Latenz berechnet?

Nach der Verbindung sendet das Tool alle 10 Sekunden eine integrierte Ping-Nachricht, misst die Zeit bis zum Empfang des Pongs und zeigt die aktuelle Latenz an.
