Preguntas frecuentes (FAQ)

¿Qué es WebSocket?

WebSocket es un protocolo que proporciona comunicación full-duplex a través de una única conexión TCP, lo que permite a los servidores enviar datos a los clientes. Esta herramienta ayuda a probar si su servicio WebSocket funciona correctamente.

¿Cómo pruebo una conexión WebSocket?

Ingrese la URL del servidor WebSocket (por ejemplo, wss://echo.websocket.org), haga clic en 'Conectar'. Una vez conectado, escriba un mensaje en el área de entrada y haga clic en "Enviar". Los mensajes recibidos aparecen en el registro.

¿Por qué falla la conexión?

Posibles motivos: dirección de servidor incorrecta, el servidor no admite WSS (pruebe con WS), bloqueo del firewall, servidor inactivo o restricciones de CORS. Verifique la URL y asegúrese de que el servidor permita conexiones WebSocket.

¿Cómo agrego encabezados personalizados?

En 'Configuración de conexión', haga clic en '+ Agregar' para ingresar el nombre y el valor del encabezado. Estos encabezados se envían durante el protocolo de enlace de WebSocket, lo que resulta útil para la autenticación o parámetros personalizados.

¿Qué formatos de mensajes se admiten?

Texto, JSON y hexadecimal. Seleccione el formato, ingrese su contenido y se convertirá automáticamente. El modo JSON incluye botones de formato y minimización para facilitar la edición.

¿Puedo guardar el registro?

Sí. Haga clic en el botón 'Exportar' en el área de registro para descargar el registro actual como un archivo de texto.

¿Cómo se calcula la latencia?

Después de la conexión, la herramienta envía un mensaje de ping incorporado cada 10 segundos y mide el tiempo hasta que se recibe el pong, mostrando la latencia actual.
