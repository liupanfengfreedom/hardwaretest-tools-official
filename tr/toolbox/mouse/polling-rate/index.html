Profesyonel Fare Olayı Frekans Test Cihazı Özellikleri

Gerçek Zamanlı Frekans İzleme

Geçerli fare olayı frekansının (Hz) gerçek zamanlı gösterimi, tarayıcının fare hareketi olaylarını işlediği gerçek hızı anlamanıza yardımcı olur.

Not: Web ortamı tarayıcı ve ekran yenileme hızıyla sınırlıdır

Jitter Analizi

Fare olaylarının zaman kararlılığını yansıtacak şekilde olay aralıklarının standart sapmasını hesaplayın. Daha düşük titreşim değerleri daha kararlı olayları gösterir.

Yüksek titreşim, fare hareketinin düzgün olmamasına neden olabilir

Maksimum Frekans Kaydı

Test sırasında elde edilen en yüksek olay sıklığını otomatik olarak kaydederek sisteminizin en iyi performansını anlamanıza yardımcı olur.

Web ortamı genellikle gerçek USB yoklama oranına ulaşamıyor

Ortalama Frekans Hesaplaması

Geçmiş verilere dayalı olarak ortalama olay sıklığını hesaplayarak daha istikrarlı performans referans göstergeleri sağlar.

Daha uzun test süresi daha doğru ortalama verir

Gerçek Zamanlı Grafik Görüntüleme

Dinamik çizgi grafikleri aracılığıyla frekans değişimi eğilimlerini görselleştirerek performans dalgalanmalarını ve anormalliklerini gözlemlemeyi kolaylaştırın.

Grafik son 100 veri noktasının eğilimlerini gösterir

CSV Verilerini Dışa Aktarma

Derin analiz ve veri koruması için tüm test günlüklerinin CSV formatında dışa aktarılmasını destekleyin.

Dışa aktarılan veriler Excel, Python vb. ile daha ileri analizler için kullanılabilir.

Sıkça Sorulan Sorular (SSS)

Web testi neden fare olay sıklığını gerçek USB yoklama oranından daha düşük gösteriyor?

Tarayıcı ortamındaki fare olayları birden fazla katmanla sınırlıdır: tarayıcı olay azaltma, ekran yenileme hızı senkronizasyonu, işletim sistemi planlama vb. Web sayfaları, yalnızca tarayıcı tarafından sağlanan fare olay arayüzü aracılığıyla USB HID katmanından ham verilere doğrudan erişemez.

Daha doğru test sonuçları nasıl alınır?

1. Tarayıcı penceresinin ön planda ve etkin olduğundan emin olun
2. Gereksiz sekmeleri ve uygulamaları kapatın
3. Hızlı, sürekli fare hareketlerini gerçekleştirintest alanı
4. Görüntüledetaylı kullanım kılavuzudaha fazla ipucu için

Jitter Nedir? Neden önemlidir?

Titreşim, fare olayı aralık sürelerinin kararlılığını yansıtır. Düşük titreşim, olay aralıklarının aynı olduğu anlamına gelir ve bu da yumuşak fare hareketi hissi sağlar. Yüksek titreşim, ortalama frekans yüksek olsa bile fare hareketinde takılma veya dengesizlik hissine neden olabilir.

Neden farklı tarayıcılar farklı test sonuçları gösteriyor?

Farklı tarayıcılar (Chrome, Firefox, Edge, Safari) farklı olay işleme mekanizmaları, kısıtlama stratejileri ve performans optimizasyonları kullanır. Chrome genellikle daha yüksek etkinlik sıklığı sağlar ancak belirli performans aynı zamanda işletim sistemi ve donanım yapılandırmasına da bağlıdır.

Fare olayı performansının normal olup olmadığı nasıl belirlenir?

Normal durumlar:
- 60Hz ekran: olay frekansı ≈ 60Hz
- 120Hz ekran: olay frekansı ≈ 120Hz
- 144Hz ekran: olay frekansı ≈ 120-144Hz
- 240Hz ekran: olay frekansı ≈ 200Hz
Frekans, ekranın yenileme hızından çok daha düşükse performans sorunları olabilir.

Fare olay sıklığını ne zaman test etmeliyim?

Önerilen test durumları:
- Fare hareketinin kekemelik veya yanıt vermeme hissi
- Farklı tarayıcılar veya sistemler arasındaki giriş performansını karşılaştırma
- Yeni ekranların veya grafik kartlarının giriş gecikmesi üzerindeki etkisini değerlendirme
- Web uygulamalarında fare yanıtı sorunlarını tanılama

Web Yoklama Oranı Testi Hakkında Önemli Notlar

Test sonuçlarını doğru bir şekilde anlamak için lütfen aşağıdaki notları dikkatlice okuyun.

Bu sayfada görüntülenen "yoklama hızı (Hz)", farenizin gerçek USB yoklama hızına eşdeğer DEĞİLDİR.

🤔 Web testi sonuçları neden genellikle yalnızca 60Hz / 120Hz?

Web tarayıcısı ortamında:

  • Fare verileri şu yolla elde edilir:tarayıcı olayları (fareyi hareket ettirme / işaretçiyi hareket ettirme)
  • Tarayıcılarkısma ve birleştirmeolaylar
  • Olay tetikleme sıklığı genellikleekran yenileme hızıyla senkronize edildi

Fare donanımınız 1000Hz USB yoklama hızına ayarlanmış olsa bile,
web ortamı, USB HID katmanından gerçek verilere doğrudan erişemez.

Gerçek 1000Hz nereye yansıyor?

Gerçek USB yoklama oranı şurada mevcuttur:

  • • İşletim sistemi ham girişi / HID katmanı
  • • Oyun motorları (Unreal Engine, Unity gibi)
  • • Masaüstü düzeyinde test araçları (web tabanlı değil)
✅
Bu araç ne için uygundur?
  • • Tarayıcıda fare girişinin düzgün olup olmadığı algılanıyor
  • • Farklı tarayıcılarda/ekranlarda etkinlik performansının karşılaştırılması
  • • Kekemelik, olay kaybı veya anormal kısıtlama kontrolü yapılıyor
❌
Bu araç ne için uygun DEĞİLDİR?
  • • Farenin 500Hz / 1000Hz'i destekleyip desteklemediğini belirleme
  • • Masaüstü USB yoklama oranı test araçlarının değiştirilmesi
