Foire aux questions (FAQ)

Qu'est-ce qu'un kit d'icônes PWA ? Pourquoi en ai-je besoin ?

A Le kit d'icônes PWA (Progressive Web App) est un ensemble de fichiers d'icônes nécessaires pour donner à un site Web l'apparence d'une application native sur les appareils mobiles. Il comprend : favicon.ico (icône de l'onglet du navigateur), favicon.svg (icône vectorielle), apple-touch-icon.png (icône de l'écran d'accueil iOS), android-chrome-192x192.png et android-chrome-512x512.png (icônes de l'écran d'accueil Android) et site.webmanifest (fichier manifeste de l'application). Les sites Web modernes ont besoin d’un kit complet d’icônes PWA pour offrir la meilleure expérience utilisateur.

Pourquoi ai-je besoin d'un Favicon au format SVG ?

SVG (Scalable Vector Graphics) Favicon présente les avantages suivants : 1) Format vectoriel, mise à l'échelle infinie sans distorsion, s'adaptant à divers appareils de résolution ; 2) Petite taille de fichier, chargement rapide ; 3) Prend en charge les effets de style et d'animation CSS ; 4) Les navigateurs modernes (Chrome, Firefox, Safari, Edge) prennent tous en charge SVG Favicon. Il est recommandé aux sites Web modernes de donner la priorité au format SVG tout en proposant le format ICO comme solution de repli pour les navigateurs plus anciens.

Pourquoi les fichiers ICO générés contiennent-ils plusieurs résolutions ?

Le format de fichier ICO prend en charge l'intégration d'images à plusieurs résolutions dans un seul fichier, appelé « ICO multi-résolution ». Les navigateurs et les systèmes d'exploitation sélectionnent automatiquement la résolution appropriée en fonction du scénario d'utilisation : 16 × 16 pixels pour les onglets du navigateur, 32 × 32 pixels pour les barres de tâches et les favoris, 48 ​​× 48 pixels pour les raccourcis du bureau. Comment déployer les icônes générées sur mon site Web ?

Le package ZIP téléchargé comprend des instructions de déploiement et des extraits de code. Étapes de base : 1) Téléchargez tous les fichiers d'icônes (.ico, .svg, .png) dans le répertoire racine de votre site Web, 2) Téléchargez également le fichier site.webmanifest dans le répertoire racine, 3) Ajoutez l'extrait de code fourni à la section <head> de votre fichier HTML. Dans la plupart des cas, il vous suffit de copier et coller le code. Le code généré par l'outil comprend déjà tous les liens et balises méta nécessaires, y compris les paramètres de priorité SVG et ICO.

Pourquoi mon site Web n'affiche-t-il pas d'icônes personnalisées sur l'écran d'accueil iOS ? Format

Si votre site Web affiche une capture d'écran de page Web au lieu d'une icône personnalisée sur l'écran d'accueil iOS, raisons possibles : 1) Le fichier d'icône Apple Touch n'est pas correctement placé dans le répertoire racine, 2) Lien d'icône Apple Touch incorrect dans le code HTML, 3) Ancienne icône mise en cache par iOS Safari. Assurez-vous que tous les fichiers sont correctement déployés et essayez d'effacer les données du site Web de Safari. Le code généré par cet outil inclut déjà tous les liens Apple-Touch-Icône nécessaires.

Quelles sont les meilleures pratiques de conception pour les Favicons ?

Lors de la conception des favicons, nous recommandons : 1) Utilisez des graphiques ou des lettres simples, évitez les détails complexes (la taille est petite), 2) Utilisez des couleurs à contraste élevé pour plus de lisibilité, 3) Conservez un design carré (les motifs circulaires s'affichent sur un fond carré), 4) Évitez les lignes fines (peuvent ne pas être visibles dans les petites tailles), 5) Testez les effets d'affichage sur différentes couleurs d'arrière-plan. La fonction de prévisualisation en temps réel de cet outil peut aider à évaluer les effets de conception.
