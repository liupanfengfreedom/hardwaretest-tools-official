Spécifications des icônes de la plate-forme iOS

  • • App Store : 1024×1024px (PNG, pas de transparence)
  • • iPhone @3x : 180 × 180 px (Retina HD)
  • • iPhone @2x : 120 × 120 px (Retina)
  • • iPad Pro : 167 × 167 px
  • • iPad @2x : 152×152px

Spécifications des icônes de la plate-forme Android

  • • Play Store : 512 × 512 px (PNG, 32 bits)
  • • xxxhdpi : 192×192px (densité extra‑extra‑extra‑haute)
  • • xxhdpi : 144×144px (densité extra‑extra‑haute)
  • • xhdpi : 96 × 96 px (très haute densité)
  • • hdpi : 72×72px (haute densité)
  • • mdpi : 48×48px (densité moyenne)
  • • ldpi : 36×36px (faible densité)

Foire aux questions (FAQ)

Pourquoi mon icône est-elle floue à certaines tailles ?

Les icônes floues sont généralement dues à une résolution insuffisante de l'image source. Nous recommandons :

  • • Utilisez une image source d'au moins 1 024 × 1 024 pixels.
  • • Préférez le vecteur (SVG) ou le PNG haute résolution
  • • Assurez-vous que les éléments clés de votre icône sont grands et clairs.
  • • Évitez les détails excessifs dans l'image source : les détails sont perdus dans les petites tailles.

Comment puis-je garantir un affichage cohérent sur différentes plates-formes ?

Les différences entre les plates-formes sont normales, mais vous pouvez les optimiser grâce à ces conseils :

  • • Conservez le contenu important dans le fichier centralzone de sécurité(éviter les bords)
  • • Utilisez des designs à contraste élevé pour une reconnaissance claire des petites tailles
  • • Vérifiez l'aperçu en temps réel pour les deux plates-formes dans le panneau de droite.
  • • iOS ajoute automatiquement les coins arrondis ; Les formes Android varient selon le thème : concevez en conséquence

Est-il compatible avec les arrière-plans transparents ?

Oui, les arrière-plans transparents sont entièrement pris en charge :

  • • Les zones transparentes dans les fichiers PNG téléchargés sont préservées
  • • Les icônes générées conservent la transparence d'origine
  • • Pour les plates-formes nécessitant un arrière-plan opaque (par exemple, iOS App Store), un arrière-plan blanc est ajouté automatiquement
  • • Considérez à quoi ressemblera la transparence sur différents arrière-plans

Quels formats de fichiers puis-je générer ?

Actuellement, les icônes sont générées principalement au format PNG :

  • • Toutes les icônes sont générées au format PNG (format multiplateforme recommandé)
  • • PNG prend en charge les arrière-plans transparents et une compression de haute qualité
  • • Pour Android, WebP peut offrir une meilleure compression mais nécessite une conversion séparée
  • • Les icônes de l'App Store iOS nécessitent un format PNG sans transparence

Cet outil est-il gratuit ? Y a-t-il des limites ?

Oui, App Icon Studio est entièrement gratuit :

  • • Utilisation gratuite, aucune inscription ni connexion requise
  • • Aucune limite d'utilisation
  • • Aucune limite de taille d'image (bien que le traitement des images volumineuses puisse prendre plus de temps)
  • • Tout le traitement s'effectue dans le navigateur, sans frais de serveur
  • • De futures fonctionnalités avancées pourront être ajoutées, mais les fonctionnalités de base resteront gratuites

Comment intégrer les icônes générées dans mon projet ?

Les étapes d'intégration varient selon la plateforme :

  • iOS (Xcode) :Faites glisser les icônes dans l'ensemble AppIcon dans Assets.xcassets
  • Android (Android Studio) :Copier les icônes dans les dossiers res/mipmap‑*dpi/
  • Réagir natif :Utilisez l'outil React‑Native‑Make ou placez-le manuellement dans les répertoires ios/ et android/
  • Flutter :Mettre à jour la configuration de l'icône dans pubspec.yaml et la placer dans les répertoires appropriés

Le ZIP généré est déjà organisé par plate-forme et par taille pour une utilisation facile.
