Fitur Penguji Frekuensi Peristiwa Mouse Profesional

Pemantauan Frekuensi Waktu Nyata

Tampilan real-time dari frekuensi kejadian mouse (Hz), membantu Anda memahami kecepatan aktual browser memproses kejadian pergerakan mouse.

Catatan: Lingkungan web dibatasi oleh browser dan kecepatan refresh tampilan

Analisis Jitter

Hitung deviasi standar interval kejadian, yang mencerminkan stabilitas waktu kejadian mouse. Nilai jitter yang lebih rendah menunjukkan kejadian yang lebih stabil.

Jitter yang tinggi dapat menyebabkan pergerakan mouse terasa tidak mulus

Perekaman Frekuensi Maksimum

Secara otomatis mencatat frekuensi peristiwa tertinggi yang dicapai selama pengujian, membantu Anda memahami kinerja terbaik sistem Anda.

Lingkungan web biasanya tidak dapat mencapai tingkat polling USB yang sebenarnya

Penghitungan Frekuensi Rata-rata

Hitung frekuensi kejadian rata-rata berdasarkan data historis, sehingga memberikan indikator referensi kinerja yang lebih stabil.

Waktu pengujian yang lebih lama memberikan rata-rata yang lebih akurat

Tampilan Grafik Waktu Nyata

Visualisasikan tren perubahan frekuensi melalui diagram garis dinamis, sehingga memudahkan untuk mengamati fluktuasi dan anomali kinerja.

Bagan menunjukkan tren dari 100 titik data terakhir

Ekspor Data CSV

Dukungan mengekspor log pengujian lengkap dalam format CSV untuk analisis mendalam dan pelestarian data.

Data yang diekspor dapat digunakan untuk analisis lebih lanjut dengan Excel, Python, dll.

Pertanyaan yang Sering Diajukan (FAQ)

Mengapa pengujian web menunjukkan frekuensi kejadian mouse lebih rendah dibandingkan tingkat polling USB sebenarnya?

Peristiwa mouse di lingkungan browser dibatasi oleh beberapa lapisan: pembatasan peristiwa browser, sinkronisasi kecepatan refresh tampilan, penjadwalan sistem operasi, dll. Halaman web tidak dapat langsung mengakses data mentah dari lapisan USB HID, hanya melalui antarmuka peristiwa mouse yang disediakan oleh browser.

Bagaimana cara mendapatkan hasil tes yang lebih akurat?

1. Pastikan jendela browser berada di latar depan dan aktif
2. Tutup tab dan aplikasi yang tidak perlu
3. Lakukan gerakan mouse yang cepat dan terus menerus di dalamarea pengujian
4. Lihatpanduan pengguna terperinciuntuk tip lebih lanjut

Apa itu Jitter? Mengapa ini penting?

Jitter mencerminkan stabilitas waktu interval peristiwa mouse. Jitter rendah berarti interval peristiwa seragam, sehingga gerakan mouse terasa mulus. Jitter yang tinggi dapat menyebabkan pergerakan mouse terasa tersendat atau tidak stabil, meskipun frekuensi rata-ratanya tinggi.

Mengapa browser yang berbeda menunjukkan hasil tes yang berbeda?

Browser yang berbeda (Chrome, Firefox, Edge, Safari) menggunakan mekanisme pemrosesan peristiwa, strategi pembatasan, dan pengoptimalan kinerja yang berbeda. Chrome biasanya menyediakan frekuensi kejadian yang lebih tinggi, namun performa spesifiknya juga bergantung pada sistem operasi dan konfigurasi perangkat keras.

Bagaimana cara menentukan apakah kinerja acara mouse normal?

Situasi normal:
- Tampilan 60Hz: frekuensi acara ≈ 60Hz
- Tampilan 120Hz: frekuensi acara ≈ 120Hz
- Tampilan 144Hz: frekuensi acara ≈ 120-144Hz
- Tampilan 240Hz: frekuensi acara ≈ 200Hz
Jika frekuensi jauh lebih rendah daripada kecepatan refresh tampilan, mungkin ada masalah kinerja.

Kapan saya harus menguji frekuensi kejadian mouse?

Situasi pengujian yang disarankan:
- Merasa gerakan mouse tersendat atau tidak responsif
- Membandingkan kinerja input antara browser atau sistem yang berbeda
- Mengevaluasi dampak tampilan atau kartu grafis baru pada latensi input
- Mendiagnosis masalah respons mouse di aplikasi web

Catatan Penting Tentang Pengujian Tingkat Polling Web

Harap baca catatan berikut dengan cermat untuk memahami hasil tes dengan benar

"Polling rate (Hz)" yang ditampilkan pada halaman ini TIDAK setara dengan polling rate USB mouse Anda yang sebenarnya.

🤔 Kenapa hasil web test biasanya hanya 60Hz / 120Hz?

Di lingkungan browser web:

  • Data mouse diperoleh melaluiacara browser (mousemove / pointermove)
  • Perambanperlambat dan gabungkanacara
  • Frekuensi pemicuan peristiwa biasanyadisinkronkan dengan kecepatan penyegaran layar
🖥️ Kecepatan Penyegaran Tampilan vs Frekuensi Pengamatan Web
Laju Penyegaran Tampilan Frekuensi Peristiwa yang Dapat Diamati di Web
60Hz ≈ 60Hz
120Hz ≈ 120Hz
144Hz ≈ 120-144Hz
240Hz ≈ 200Hz

Bahkan jika perangkat keras mouse Anda disetel ke tingkat polling USB 1000Hz,
lingkungan web tidak dapat secara langsung mengakses data nyata dari lapisan USB HID.

Di mana 1000Hz yang sebenarnya tercermin?

Tingkat polling USB asli ada di:

  • • Input mentah sistem operasi / lapisan HID
  • • Mesin game (seperti Unreal Engine, Unity)
  • • Alat pengujian tingkat desktop (bukan berbasis web)

  • • Memeriksa kegagapan, kehilangan peristiwa, atau pelambatan yang tidak normal

  • • Menentukan apakah mouse mendukung 500Hz / 1000Hz
  • • Mengganti alat pengujian tingkat polling USB desktop
