Varian Hash #explain-variant
MD5-32
Keluaran Standar Format MD5 standar yang ditentukan dalam RFC 1321 menghasilkan a128-bitdigest untuk input dengan panjang berapa pun dan menampilkannya sebagai32karakter heksadesimal huruf kecil. Ini banyak digunakan untuk pemeriksaan integritas file, sidik jari data, dan deduplikasi. Contoh:5d41402abc4b2a76b9719d911017c592
MD5-16
Keluaran Terpotong Mengambil karakter9–24dari output standar MD5-32 (total 16 karakter), secara efektif mengekspos 64 bit tengah. Ini lebih pendek, tetapi memiliki risiko tabrakan yang lebih tinggi (kira-kira2⁻⁶⁴). Cocok untuk ID pendek atau token dalam skenario keamanan rendah, namun tidak disarankan untuk validasi keamanan.
MD5-32 Huruf besar
Keluaran Huruf Besar Nilai hash yang sama persis dengan MD5-32, tetapi dengan huruf besar (A–F). Beberapa sistem lama, tanda tangan pembayaran, atau antarmuka perusahaan memerlukan keluaran huruf besar. Contoh:5D41402ABC4B2A76B9719D911017C592
📥
Pengkodean Masukan #explain-encoding
UTF-8 Teks
Bawaan Memperlakukan masukan sebagai aUTF-8string yang dikodekan dan secara otomatis mengonversi karakter multibyte seperti China, Jepang, atau emoji menjadi urutan byte sebelum melakukan hashing. Ini adalah mode paling umum dan berfungsi untuk input teks normal.
Aliran Byte Hex
Heksadesimal Memperlakukan input sebagai string heksadesimal (0-9, a-f, spasi diabaikan). Setiap dua karakter diurai sebagai satu byte sebelum hashing. Ini berguna saat melakukan hashing pada data biner mentah seperti kunci atau muatan protokol. Contoh masukan:48 65 6c 6c 6f
Base64
Base64 Dekode Memperlakukan masukan sebagai standarBase64string, mendekodekannya menjadi byte mentah, dan melakukan hash pada byte tersebut. Berguna saat menghitung hash asli konten file atau gambar yang direpresentasikan di Base64. Contoh:SGVsbG8=
📋
Mode Batch #explain-batch
Teks Tunggal / Lengkap
Bawaan Menghash seluruh konten kotak masukan, termasuk jeda baris, sepertisatu nilai lengkapdan menampilkan satu hasil. Cocok untuk sidik jari teks lengkap, paragraf, atau konten file. Jeda baris disertakan dalam input hash.
Dibagi per Baris
Batch Membagi input menjadi beberapa baris menggunakan\nsebagai pemisah dan melakukan hash pada setiap baris secara independen. Baris kosong diabaikan secara otomatis. Output utama menunjukkan satu hash per baris, dan tabel batch lengkap di bawah mencakup MD5-32, MD5-16, dan hasil huruf besar dengan satu klikCSVekspor. Berguna untuk nama pengguna, daftar kata sandi, dan pemrosesan kumpulan data.
