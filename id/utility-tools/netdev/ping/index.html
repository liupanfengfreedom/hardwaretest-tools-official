Fitur Diagnostik Jaringan Profesional

📖 Pendahuluan:

Uji Mode Ganda

ICMP Ping (proxy jarak jauh) dan HTTPS GET (browser langsung) berdampingan, memenuhi berbagai kebutuhan diagnostik.

Proxy Ping melewati batasan browser; HTTPS langsung mencerminkan pengalaman pengguna yang sebenarnya. Analisis jaringan dari lapisan jaringan dan lapisan aplikasi – ideal untuk DevOps, pemantauan situs web, analisis latensi game.

Visualisasi Waktu Nyata

Bagan garis dinamis perubahan latensi, permintaan berhasil/gagal dengan kode warna.

Bagan berskala otomatis, menyoroti anomali. Temukan periode latensi puncak dengan cepat dan optimalkan jaringan Anda.

Sejarah Cerdas

Secara otomatis menyimpan target yang baru diuji; klik kolom masukan untuk memilih satu, hapus satu klik.

Riwayat disimpan secara lokal, tidak pernah diunggah. Gunakan kembali target yang sering, tingkatkan produktivitas.

Log Terminal

Setiap urutan permintaan, waktu respons, status (HTTPS), atau indikasi kehilangan tercetak dengan jelas.

Hingga 30 entri, dapat digulir. Telusuri setiap detail untuk mendiagnosis masalah yang sewaktu-waktu terjadi.

Perhitungan Jitter

Secara otomatis menghitung perbedaan absolut rata-rata antara penundaan berturut-turut – metrik stabilitas yang tepat.

Jitter >30ms dapat memengaruhi komunikasi real-time (VoIP, video). Pantau jitter untuk memandu penyetelan jaringan.

Mode Berkelanjutan

Pilih “Terus menerus” untuk mengirim permintaan hingga dihentikan secara manual – ideal untuk pemantauan stabilitas jangka panjang.

Klik “Hentikan Tes” (berubah menjadi merah setelah dimulai) untuk menyela. Sempurna untuk menangkap periode fluktuasi.

Pertanyaan yang Sering Diajukan (FAQ)

Bagaimana cara menguji server atau situs web saya sendiri?

Masukkan domain atau IP Anda (misalnya, example.com atau 93.184.216.34) di bidang target, pilih mode pengujian (Ping atau HTTPS), dan klik Mulai. Untuk pengujian HTTPS, pastikan server Anda mendengarkan pada port 443.

Mengapa latensi Ping dan HTTPS berbeda?

ICMP Ping mengukur waktu bolak-balik lapisan jaringan, sedangkan HTTPS GET melibatkan jabat tangan TCP, negosiasi TLS, dan permintaan/respons HTTP – biasanya memerlukan waktu puluhan milidetik lebih lama, dan ini merupakan hal yang diharapkan.

Berapa tingkat kehilangan paket yang dianggap tidak normal?

Untuk jaringan kabel, kerugian harusnya 0% sepanjang waktu; untuk Wi‑Fi, hingga 0,1% dapat diterima. Kehilangan yang konsisten di atas 0,5% atau seringnya waktu habis menunjukkan masalah yang signifikan – periksa jaringan lokal Anda atau hubungi ISP Anda.

Mengapa Anda menggunakan proxy jarak jauh untuk Ping?

Browser tidak dapat mengirim paket ICMP mentah karena batasan keamanan. Alat ini menggunakan proksi backend (atau API publik) untuk melakukan Ping dan mengembalikan hasilnya, sehingga memungkinkan pengujian ICMP ke target mana pun.

Apakah lokasi server proxy mempengaruhi hasil Ping?

Ya – Latensi Ping ICMP mencakup waktu dari proxy ke target, bukan dari lokasi Anda. Namun, loss dan jitter tetap mencerminkan stabilitas jaringan target. HTTPS GET tidak terpengaruh karena dijalankan langsung dari browser Anda.

Mengapa HTTPS terkadang menampilkan "blok lokal"?

Jika permintaan gagal dalam waktu 5 ms, biasanya permintaan tersebut diblokir oleh ekstensi browser (misalnya, pemblokir iklan). Alat ini menandainya sebagai “Blok Lokal” dan bukan kegagalan jaringan. Coba nonaktifkan ekstensi dan tes ulang.

Berapa banyak data yang digunakan mode berkelanjutan?

Setiap permintaan Ping hanya berukuran puluhan byte; Permintaan HTTPS ringan (HEAD/no‑cors). Bahkan setelah satu jam, total lalu lintas biasanya di bawah 10 MB – aman untuk digunakan.
