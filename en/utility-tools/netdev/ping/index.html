Frequently Asked Questions (FAQ)

What is the difference between ICMP Ping and HTTPS GET tests?

ICMP Ping sends ICMP packets through a remote proxy server to measure network layer connectivity and latency, suitable for checking if a target is online and basic network quality. HTTPS GET is directly initiated by your browser to measure application layer response time, better reflecting the actual experience of visiting a webpage or API.

Why is packet loss not zero?

Packet loss may be caused by network congestion, faulty routing nodes, target server throttling, or unstable local network. Persistent non-zero loss usually indicates potential network quality issues.

What impact does high jitter have?

Jitter refers to the variation in latency. High jitter can cause stuttering or audio/video sync issues in real-time applications (e.g., video conferencing, online gaming). Typically, jitter should be below 30ms.

How do I test my own server or website?

Enter your domain or IP in the target input (e.g., example.com or 93.184.216.34), select test mode (Ping or HTTPS), and click Start Test. For HTTPS test, ensure your server supports port 443.

Why are ICMP Ping and HTTPS GET latencies different?

ICMP Ping measures network layer round-trip time, while HTTPS GET requires TCP handshake, TLS negotiation, and HTTP request/response, which takes longer. It's normal for HTTPS latency to be tens of milliseconds higher than Ping.

What packet loss rate is considered abnormal?

For wired networks, packet loss should be 0% over time; for wireless, occasional 0.1% is acceptable. If loss consistently exceeds 0.5% or timeouts occur frequently, it indicates significant network issues‚Äîcheck your local network or contact your ISP.

Why use a remote proxy for Ping?

Due to browser security policies, web pages cannot send raw ICMP packets. This tool uses a backend proxy (or public API) to send Ping requests on behalf of the browser and returns the results, enabling ICMP testing to any target.

Are test results affected by the proxy server location?

Yes, ICMP Ping latency includes the time from the proxy server to the target, so it doesn't represent your local latency to the target. However, loss and jitter still reflect the target network's stability. HTTPS GET is directly from your browser and is not affected.

Why does HTTPS test sometimes show 'local block'?

If a request fails within 5ms, it's usually blocked by browser extensions or plugins (e.g., ad blockers). The tool marks it as 'local block', not a network failure. Try disabling extensions and test again.

How much data does continuous test mode consume?

Each Ping request is only tens of bytes, and HTTPS requests are lightweight (HEAD/no-cors). Even after an hour of continuous testing, total traffic is typically under 10MB, so feel free to use it.
