Frequently Asked Questions (FAQ)

What is the WebSocket protocol?

WebSocket is a protocol for full-duplex communication over a single TCP connection. It simplifies data exchange between clients and servers, allowing servers to actively push data to clients. Unlike HTTP, once a WebSocket connection is established, it remains open continuously, avoiding the repeated connection overhead of HTTP.

How do I test a local WebSocket server?

If your WebSocket server is running locally, use the following address format:
ws://localhost:port or ws://127.0.0.1:port
Example: ws://localhost:8080 or ws://127.0.0.1:3000

Why does the connection fail?

Possible reasons for connection failure:
1) Incorrect server address or server not running
2) Network firewall blocking WebSocket connections
3) Server doesn't support WebSocket protocol
4) Wrong protocol used (should use wss:// instead of ws://, or vice versa)
5) Cross-origin access restrictions
Recommended checks: server status, network connection, and protocol settings.

How do I save test results?

The tool currently supports automatic saving of connection history. Log content can be viewed through the browser console or manually copied for saving. Future versions will add log export functionality, supporting export of test results in JSON or text format.

Which WebSocket subprotocols are supported?

This tool supports the standard WebSocket protocol. If specific subprotocols are needed, you can add protocol parameters after the connection address, or contact the development team for custom functionality. Common WebSocket subprotocols include STOMP, MQTT, etc.
