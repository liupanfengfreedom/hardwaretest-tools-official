Frequently Asked Questions (FAQ)

Which audio formats are supported?

Currently we support conversion between: MP3, WAV, FLAC, AAC (M4A), OGG (Opus), AIFF, WMA, WebM (Opus). You can also extract audio from video files (MP4, MKV, MOV, AVI, 3GP, TS) and convert it to any of these formats. More formats will be added over time.

How good is the converted audio quality?

Quality depends entirely on your settings:
- For WAV or AIFF: lossless PCM, identical to the original audio data.
- For MP3: use the slider to set bitrate (64 kbps to 320 kbps) ‚Äì higher gives better quality and larger files.
- Sample rate options let you downsample to reduce file size (e.g., 48kHz ‚Üí 44.1kHz or 22kHz).
We recommend keeping original sample rate and stereo for best results.

Are files really not uploaded?

Yes, 100% local processing. We use the Web Audio API to decode, lamejs to encode MP3, and MediaRecorder for Opus. Everything happens in your browser's memory ‚Äì no network transfer. You can even use this tool offline (after the page has been loaded once).

Why can't I convert some video files?

Browsers can only decode video codecs they support. For instance, some MKV files may contain HEVC (H.265) video, and Safari or Chrome might not support its audio track. Try using a more common codec (like H.264 + AAC). If problems persist, convert the video encoding first, then extract the audio.

Can I convert multiple files at once?

Yes. Select several files ‚Äì they appear in the queue. Click ‚ÄúStart Conversion‚Äù and the tool processes them one by one. You can add or remove files during conversion, but files that are currently being processed cannot be modified.

Will the page freeze when converting large files?

Audio encoding is CPU‚Äëintensive, especially for large files or high bitrates. Although we optimise as much as possible, the main thread may briefly block. For very large files (e.g., >100 MB), please be patient ‚Äì the progress bar updates in real time. A future version may introduce Web Workers for smoother performance.
