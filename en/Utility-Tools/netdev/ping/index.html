Frequently Asked Questions (FAQ)

What is Ping latency?

Ping latency refers to the time taken for a data packet to travel from your device to a target server and back, typically measured in milliseconds (ms). Lower latency means faster network response. Generally: <50ms excellent, 50-100ms good, 100-200ms fair, >200ms poor.

Why test packet loss?

Packet loss indicates the percentage of data packets lost during transmission. Packet loss causes unstable network connections, affecting real-time applications like video calls and online gaming. Ideally packet loss should be 0%, below 2% is acceptable, above 5% may indicate network issues.

How does network jitter affect applications?

Jitter refers to variation in network latency. High jitter causes stuttering in voice/video calls, latency fluctuations in online games, and unstable real-time data transmission. For VoIP and video conferencing, jitter should be below 30ms; for online gaming, below 50ms.

How to interpret test results?

A good network connection should have: low latency (<100ms), low packet loss (<2%), low jitter (<30ms). If results are poor, try: restarting your router, changing DNS servers, avoiding peak network hours, contacting your internet service provider.

Can I test local networks?

Yes, you can test local network devices like routers (typically 192.168.1.1 or 192.168.0.1). Local network latency should be very low (<5ms). High latency may indicate LAN issues.
