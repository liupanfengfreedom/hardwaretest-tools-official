Frequently Asked Questions (FAQ)

What is WebSocket Protocol?

WebSocket is a protocol for full-duplex communication over a single TCP connection. It makes data exchange between client and server simpler, allowing servers to actively push data to clients. Unlike HTTP, once a WebSocket connection is established, it can remain connected continuously.

How to test a local WebSocket server?

If your WebSocket server runs locally, you can use the following address format:
ws://localhost:port or ws://127.0.0.1:port
For example: ws://localhost:8080 or ws://127.0.0.1:3000

Why does connection fail?

Possible reasons for connection failure include:
1) Wrong server address or server not running
2) Network firewall blocking WebSocket connection
3) Server doesn't support WebSocket protocol
4) Using wrong protocol (should use wss:// instead of ws://, or vice versa)
Recommended to check server status, network connection, and protocol settings.

How to save test results?

The tool currently supports automatic saving of history connection records. Log content can be viewed through browser console, or manually copied and saved. Future versions will add log export functionality.

Which WebSocket subprotocols are supported?

This tool supports standard WebSocket protocol. If specific subprotocols are needed, you can add protocol parameters after the connection address, or contact the development team for custom features.
